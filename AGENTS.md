# Argus — AGENTS.md

Fully implemented research agent (60+ source files, 211 tests, evaluation pipeline). Design docs (`project.md`, `PLAN.md`) are the architecture source of truth but are aspirational/historical — always verify against actual code under `argus/`.

## Entry point & commands

| Command | What it does |
|---------|-------------|
| `python -m argus` | Starts server + all agent workers + KG writer (threads) |
| `python -m argus --workers-only` | Workers only (pair with `uvicorn argus.app:app --reload --port 8001` for hot-reload) |
| `python -m argus onboard` | Interactive LLM/search provider setup wizard |
| `python -m argus research "query" [--max-sources N] [--time-limit N]` | Submit research via CLI, watch SSE progress |
| `python -m argus list` | List all research tasks |
| `python -m argus status <task_id>` | Show task status |
| `python -m argus profile` | Interactive stage-by-stage provider/model assignment |
| `python -m argus profile-list` | Show current stage profile |
| `python -m argus profile-clear` | Clear all stage profile assignments |
| `python -m argus search` | Interactive search provider priority & settings |
| `python -m argus search-list` | Show current search provider config |

## Developer conventions

- **Cost tracker** on every LLM call and tool call (`services/tools/cost_tracker.py`)
- **Budget enforcer** hard stop at $0.50/task — raises `BudgetError` before overshoot
- **LLM response cache** wired into router (`llm/router.py` — `LLMCache` lookup before provider call)
- **Source cache** wired into DeepDiveAgent (`services/agents/deep_dive.py` — `SourceCache` check before scrape)
- **Verification capped** at 10 claims per entity to prevent O(n²) explosion
- **Retry + circuit breaker** on every external call (tenacity + pybreaker)
- **No direct SQLite writes from agents** — always emit facts to Redis `facts` stream; KG Writer batch-inserts
- **Idempotency key** (UUID v7) on every Redis stream message
- **Agent runners** consume from `tasks:{agent_type}` stream, each agent type has its own consumer group
- **Provider config** persisted at `~/.argus/providers.json` (generated by `onboard` wizard), fallback to env defaults
- **Config** via env vars with `ARGUS_` prefix (e.g. `ARGUS_REDIS_URL`, `ARGUS_GROQ_API_KEY`)
- **LLM router** routing table (hardcoded in `llm/router.py:41-101`):

| Task | Primary | Fallback chain |
|------|---------|---------------|
| planning | Ollama | Groq → OpenRouter → OpenAI → Anthropic → Google AI Studio → DeepSeek → Together AI → LiteLLM |
| scout | Groq | Ollama → OpenRouter → OpenAI → Google AI Studio → DeepSeek → Together AI |
| deep_dive | Ollama | Groq → OpenRouter → OpenAI → Anthropic → Google AI Studio → DeepSeek → Together AI → LiteLLM |
| verification | Groq | Ollama → OpenRouter → OpenAI → Anthropic → Google AI Studio → DeepSeek |
| synthesis | Ollama | Groq → OpenRouter → OpenAI → Google AI Studio → DeepSeek → Together AI → LiteLLM |
| conflict_resolution | Groq | OpenRouter → OpenAI → Anthropic → Google AI Studio → DeepSeek → OpenAI-Compatible |

**Stage profiles** (`~/.argus/stage_profiles.json`) can override these defaults per task type. If set, the assigned provider+model is tried first before the default fallback chain. Configure via `python -m argus profile`.

**Search priority** (`~/.argus/providers.json`) determines the fallback order for web searches. The `WebSearch` class reads enabled providers sorted by priority (lowest first) and tries each in sequence. Configure via `python -m argus search` or during `onboard`. Falls back to env-var-based config when no saved search providers exist.

## Pipeline improvements (June 2026)

### LLM planner (now default)
`ResearchManager` uses `LLMPlanner` (`orchestrator/planner/llm_planner.py`) by default instead of `RuleBasedPlanner`. The LLM dynamically decomposes each query into steps (discover, extract, verify, synthesize). Falls back to `RuleBasedPlanner` on failure.

### Synthesis step in every plan
Both `LLMPlanner` and the fallback `RuleBasedPlanner` now include a `SYNTHESIZE` step as the final pipeline stage. `SynthesisAgent` (`services/agents/synthesis.py`) extends `BaseAgent` — its `run()` method queries the KG for entities/claims and generates a cohesive markdown synthesis via LLM. The agent also continues its existing role as a stream consumer for entity merging.

### Query context flows to all agents
The original research query is passed through the Redis message → `TaskStep.query` → every agent. Agent prompts now include the query context:
- **Deep dive**: Prompt begins with `Research context: {query}` before sources
- **Verification**: Research context added to conflict check prompt
- **Scout**: LLM analyzes search results against the query, extracts entities with relevance filtering
- **Synthesis**: Query used in the final summary prompt

### Scout LLM-powered
Scout (`services/agents/scout.py`) now calls the LLM after search to analyze results against the query. It classifies relevance (high/medium/low) and extracts entities with descriptions. Falls back to raw results if the LLM call fails.

### Larger content window
Deep dive content per source increased from 2000 to 8000 chars (`MAX_CONTENT_CHARS` in `deep_dive.py`).

## Tech stack

| Layer | Choice |
|-------|--------|
| Lang | Python >=3.11 |
| Web | FastAPI + uvicorn |
| Queue | Redis Streams (redis-py + hiredis) |
| KG | SQLite + JSON cols + recursive CTEs (no Neo4j) |

| Scrape | httpx+BS4 > Playwright > Firecrawl |
| PDF | pymupdf |
| Lint | ruff (target py311, line-length 100) |
| Type | mypy --strict (excludes tests/) |
| Test | pytest, pytest-asyncio (auto mode), pytest-cov |
| Infra | Docker Compose (Redis + app) |

## Key constraints (from design but verified in code)

- **Single-user local** — no HA, no K8s. Docker Compose is the deployment model.
- **< $0.50 per research** — free-first tools, aggressive caching, cost-aware routing, budget enforcer.
- **Evaluation** fully implemented at `evaluation/` with 3 datasets (market_research, tech_comparison, academic_survey), metrics, ablation, and runner.
